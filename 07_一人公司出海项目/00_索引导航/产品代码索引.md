## 五、单产品标准结构（以 ai-fitness-plan-generator 为例）

```
12_Micro_SaaS出海/ai-fitness-plan-generator/
├── package.json          # 依赖与脚本
├── next.config.js        # Next.js 配置
├── vercel.json           # Vercel 配置（如有）
├── pages/
│   ├── index.js          # 首页
│   ├── _app.js           # App 壳
│   └── api/              # API 路由（Pages Router 格式）
├── src/                  # 业务逻辑
├── public/               # 静态资源
└── node_modules/         # 安装依赖（不复制）
```

> 路由冲突根因：同时存在 `pages/`（Pages Router）和 `app/` 或 `src/app/`（App Router）。新开发请**只用一种**，推荐 Pages Router（与现有部署脚本一致）。
