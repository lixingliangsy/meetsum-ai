# 流水线手册 · 13 个脚本详解

## 一、部署类

### 1. `auto_deploy_all.py` — 批量部署全部产品到 Vercel
- **功能**：遍历 `12_Micro_SaaS出海` 下的所有产品目录，逐个执行 `vercel --prod --yes`，解析输出中的生产 URL 并记录。
- **调用**：`python auto_deploy_all.py`
- **依赖**：已登录的 Vercel CLI（`vercel login`）
- **输入输出**：输入=产品目录；输出=控制台 URL 列表 + 部署日志
- **跨机修复**：第 20 行
  ```python
  BASE_DIR = r"e:\AgentCPM\07_一人公司出海项目\12_Micro_SaaS出海"  # ← 改成目标机器路径
  ```
  第 37 行 `subprocess.run(["vercel", "--prod", "--yes"], cwd=product_dir)` —— 若 `vercel` 不在 PATH，改用绝对路径（参考 batch_deploy.py 写法）。

### 2. `batch_deploy.py` — 批量部署（带 vercel.cmd 绝对路径）
- **功能**：与 `auto_deploy_all.py` 类似，但使用 WorkBuddy 托管的 `vercel.cmd` 绝对路径，规避 Windows 上 Python 找不到 `vercel` 的问题。
- **调用**：`python batch_deploy.py`
- **依赖**：WorkBuddy 安装的 Node 22 下的 `vercel.cmd`
- **跨机修复**：
  - 第 7 行 `products_dir = "e:/AgentCPM/07_一人公司出海项目/12_Micro_SaaS出海"` → 改目标路径
  - 第 48 行 `vercel_cmd = r"C:\Users\lixingliang\.workbuddy\binaries\node\versions\22.22.2\vercel.cmd"` → 改目标机器的 vercel 路径，或改为 `"vercel"`

### 3. `batch_deploy.sh` — Shell 版批量部署
- **功能**：Bash 循环部署产品列表。
- **调用**：`bash batch_deploy.sh`
- **依赖**：`vercel` 在 PATH

### 4. `deploy_all_products.bat` — Windows 批处理部署
- **功能**：Windows 下一键触发部署流程。
- **调用**：`deploy_all_products.bat`
- **依赖**：同 vercel CLI

## 二、状态/验证类

### 5. `monitor_vercel_deployments.py` — 监听 Vercel 部署状态
- **功能**：通过 Vercel API 轮询部署状态。
- **调用**：`python monitor_vercel_deployments.py`
- **依赖**：`VERCEL_TOKEN` 环境变量（已可配置 ✅，无需硬编码）
- **跨机**：只需在目标机器 `export VERCEL_TOKEN=xxx` 即可复用。

### 6. `check_products_status.py` — 检查产品部署/可访问状态
- **功能**：批量检查各产品 URL 可访问性。
- **调用**：`python check_products_status.py`
- **依赖**：`requests`
- **注意**：沙箱网络无法访问 `*.vercel.app` 子域名时会超时，需在能联网的机器运行。

### 7. `verify_product_features.py` — 核实产品功能真实性
- **功能**：检查产品是否真正实现了 FAQ/文章中宣传的功能，输出核对报告。
- **调用**：`python verify_product_features.py`
- **依赖**：无

### 8. `product_access_check.py` — 产品可访问性检查
- **功能**：HTTP 层面检查已部署产品是否可访问。
- **调用**：`python product_access_check.py`
- **依赖**：`requests`
- **注意**：同 check_products_status.py，沙箱网络受限时会超时。
