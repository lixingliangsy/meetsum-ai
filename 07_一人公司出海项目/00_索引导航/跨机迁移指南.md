# 跨机迁移指南 · 复制到另一台电脑并行开发

> 适用场景：将 `E:\AgentCPM\07_一人公司出海项目` 整包复制到另一台电脑，继续产品开发与部署。
> 目标：零踩坑复用流水线与代码。

## 一、打包前排除清单（减小体积 + 避免泄露密钥）

用以下 `.gitignore`（仓库已包含）或压缩时排除：

```
node_modules/
.next/
.vercel/
*.log
build_*.log
.env.local
.env.production
.env
```

**务必排除的理由**：
- `node_modules/`（每台机器 `npm install`，跨机复制巨大且无意义）
- `.next/` `.vercel/`（构建缓存 + 部署配置，目标机器重新生成）
- `.env.local` / `.env.production`（**含 Waffo/LinkedIn/Vercel 真实密钥，勿明文传**）


## 六、并行开发建议（两台电脑同时开发）

1. **统一用 Git 管理**：两台机器都从同一远程仓库拉取/推送，用分支隔离（如 `dev-pc-a` / `dev-pc-b`），避免互相覆盖。
2. **`.env.local` 各机独立**：密钥文件不入库，各自本地维护。
3. **`12_Micro_SaaS出海` 为唯一真相源**：顶部 5 个同名副本不要两边都改，避免分叉。建议在另一台电脑只保留 `12_Micro_SaaS出海` 即可。
4. **路由规范**：新开发只使用 **Pages Router**（`pages/`），不要混用 `app/`，否则部署会触发路由冲突（见产品代码索引第四节）。
5. **部署分工**：两台机器可分别部署不同产品，但 Vercel 项目名需唯一，避免同项目并发部署冲突。
