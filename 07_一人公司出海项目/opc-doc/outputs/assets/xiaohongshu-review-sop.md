# 净文 AI · 小红书周度复盘 SOP（单人运营版）

> 配套：`asset-register.md`（资产登记表）｜适用：单人 / 小团队运营净文 AI 小红书账号
> 目的：把每周运营数据变成**可决策的下一步动作**，并持续反哺选题库与资产库。
> 合规底线：本 SOP 涉及的一切内容动作，均须遵守「不绕检测、不代写、AI 参与必标注」。

---

## 1. 复盘节奏

| 项 | 建议 |
|---|---|
| 频率 | **每周一次**（建议周日 21:00–22:00，承接本周数据、规划下周选题） |
| 耗时 | **45–60 分钟**（冷启动前 3 周可加密：周三 + 周日各一次快复盘，每次 20 分钟） |
| 输入 | 小红书创作中心本周数据 + 官网 Supabase 后台 `profiles`/`documents`/`subscriptions` 表 |
| 输出 | 本周决策（保留/换钩子/回看定位）+ 下周选题排期 + 资产库新增回填 |
| 工具 | 一张表格（下面模板）+ 创作中心导出 + Supabase SQL 查询 |

## 2. 必看指标 + 健康阈值（冷启动期参考）

| 指标 | 口径 | 健康阈值 | 预警线 |
|---|---|---|---|
| 单篇 7 日曝光 | 创作中心单篇数据 | 500–2000 爬坡；>3000 小爆 | <200 查标题/封面/限流 |
| 点击率 CTR | 点击 ÷ 曝光（封面吸引力） | ≥3% 健康；≥5% 优秀 | **<2% 换封面/钩子** |
| 收藏率 | 收藏 ÷ 曝光（干货价值信号） | ≥3% | <1% 内容干货不足 |
| 互动率 | (赞+藏+评+转) ÷ 曝光 | ≥5% | <2% 缺钩子/话题 |
| 涨粉 | 单篇带来关注 / 周净增 | 引流款单篇 +5~20；周净增 ≥10 | **连续 3 篇涨粉<5 回看定位** |
| 官网引流 | 经主页/置顶点击链接的访客数 | 随发布日波动、发布后 24h 起峰 | 发布后无峰值→引导语失效 |
| 试用转化 | 免费 3 篇体验注册数 ÷ 引流访客 | **>3%**（首月 KPI） | <3%→优化落地页/首屏钩子 |
| 付费转化 | Pro 注册 ÷ 免费体验用户 | 观察为主（Waffo 后台看） | 注册多付费低→查价值感知/pricing |

> 说明：阈值为单人冷启动经验值，跑满 4 周后用自己的基线校准（用前 4 周均值做"个人基准线"替代本表默认值）。

## 3. 数据来源

**A. 小红书侧（创作中心）**
- PC 端创作者中心 / App「创作中心」→ 单篇数据、7 日趋势、粉丝来源。
- 记录字段：曝光、点击、CTR、赞/藏/评/转、涨粉、粉丝来源词。

**B. 官网侧（Supabase 后台）**
- `profiles` 表：`created_at`（注册时间，用来看引流高峰是否跟发布节奏一致）、`plan`（free/pro）、`documents_used`、`waffo_customer_id`（非空=付费）。
- `documents` 表：`created_at`、`user_id`、`status`、`char_count`（看免费体验是否被使用）。
- `subscriptions` 表：`plan`、`status`（active=付费转化，来自 Waffo webhook）。
- 关联方法：把小红书「发布日」与 `profiles.created_at` 周趋势叠加，验证"发布→官网注册"的因果节奏。

**C. 快捷查询（Supabase SQL Editor 示例）**
```sql
-- 本周新增注册（看引流转化）
select count(*) from profiles
where created_at >= now() - interval '7 days';

-- 本周付费转化
select count(*) from subscriptions
where status = 'active'
  and current_period_start >= now() - interval '7 days';

-- 免费体验是否被用（documents_used>0 比例）
select
  count(*) filter (where documents_used > 0) as used,
  count(*) as total
from profiles;
```

## 4. 决策规则（if → then）

| 触发条件 | 决策动作 |
|---|---|
| 某选题 **CTR < 2%** | 换封面类型 / 标题钩子（同内容用 `generate_carousel.py` 重出图做 A/B） |
| 收藏率 **< 1%** | 内容干货密度不足，补"可收藏清单 / 对比表 / 一图流" |
| **连续 3 篇涨粉 < 5** | 回看账号定位与内容支柱配比（引流 60/专业 30/人设 10 是否跑偏） |
| 曝光长期 **< 200** | 查是否被限流（复核合规护栏）、标题是否没埋搜索词 |
| **引流高但转化低**（官网访客多、注册<3%） | 优化落地页首屏价值钩子、免费体验引导、CTA 文案 |
| 注册多但**付费低** | 查 Pro 价值感知、pricing 页、Waffo 流程是否顺畅 |
| 某封面类型 CTR 持续高于均值 | 固化为"高点击封面模板"，写入资产登记表 |

> 原则：先调**单一变量**（一次只换封面或只换标题），才能归因；禁止同时改多处后无法判断有效因素。

## 5. 选题库迭代方式

- **表现好 → 扩成系列**
  - 判定：CTR ≥5% **或** 单篇涨粉 ≥20 **或** 进入搜索长尾词前排。
  - 动作：在同一内容支柱下再写 2–3 篇变体（同主题不同角度），回写 `asset-register.md` 选题库，标记为「系列」。
- **表现差 → 归档待优化**
  - 判定：CTR <2% 连续 2 篇，或收藏率 <1% 且无涨粉。
  - 动作：保留选题，移入"待优化"区，仅**换钩子/封面**重测，不轻易废题（长尾词可能只是时机问题）。
- **每月补充**
  - 用小红书搜索框挖「降 AI 率 / 论文降重 / AI 标识」下 10 个高频问题，补充进选题库（呼应 launch plan 第 3 节搜索流量打法）。
- **淘汰**
  - 连续 2 个月无起色且无搜索量 → 移出活跃库，存入"归档"备查。

## 6. 合规自检清单（每次发帖前勾选）

> 任何一项未过，不发。这是净文 AI 的底线，也是《人工智能生成合成内容标识办法》+ 小红书治理红线的要求。

- [ ] **AI 标识**：凡 AI 参与生成的笔记，已勾选「笔记含 AI 合成内容」（设置 → 内容类型声明）
- [ ] **未绕过检测**：内容未承诺 / 暗示"绕过 AI 检测""不标注也能过审"
- [ ] **未代写**：未提供 / 鼓励替他人代写、未把净文 AI 包装成"代写工具"
- [ ] **非冒充真人**：未用 AI 虚拟人设伪装真人；人设款为真实运营者
- [ ] **无违禁夸大词**：未使用"最好/第一/顶级/绝对/100%有效"等词（已用违禁词检测过）
- [ ] **引流合规**：仅通过主页简介 / 置顶评论引导（未私信硬广、未留外链/微信号）
- [ ] **案例脱敏**：涉及用户案例已脱敏且获授权

## 7. 与资产登记表的联动

- 每周复盘末尾花 **5 分钟**做判断：**本周是否有可固化成果？**
- 可固化信号举例：
  - 新发现的高点击封面样式 → 回填 `asset-register.md` 资产 2（封面模板），更新"复用方式"。
  - 跑通的爆款选题 → 回填资产 1（选题库），标记系列。
  - 新的合规引流话术（A/B 胜出） → 建议新增"引流话术库"资产（见资产登记表「建议 B」）。
  - 新的改写前后对比案例 → 建议沉淀"案例库"资产（见资产登记表「建议 A」）。
- 联动纪律：新增可复用素材**当天回填** `asset-register.md`，保持资产表与运营同步，不积压。
