### 1.1 业务定位
- **营收引擎（执行期）**：**PurePen AI（净文 AI）**——AI 降重 / 降 AI 率 SaaS，订阅制，合规条款 D-15。

### 1.2 流量漏斗（H4 · 零凭证起量）
- 模型：`自然曝光 → 点击 → 咨询 → 留资`
- 阈值（占位校准值）：7 日曝光均 ≥ 30 / 咨询 7 日合计 ≥ 1 / 留资率 ≥ 1%

### 1.3 变现（H0 闸门）
- Waffo / Stripe 仅 `payment_config/` 模板就绪；凭证未填、未部署。
- 触发条件：H4 连续 3 天全达标 → 生成"🚦 H0 上线建议"，仍需人工确认凭证 + Vercel 部署（不自动烧凭证）。

### 1.4 技术与部署
- GitHub 110 仓库（TypeScript 为主 / 健身类为 HTML），本地 `12_Micro_SaaS出海/` 为副本。
- **技术债**：3 个产品"构建中(待确认)"（calorie-tracker / ai-fitness-planner / fitness-challenge-tracker）；1 个"源码丢失"跳过（ai-changelog-generator）；`waffo-automation/` `browser-automation/` 仅含 node_modules；部署脚本（`auto_deploy_all.py` L20、`batch_deploy.py` L7/L48）**硬编码绝对路径 + vercel.cmd 路径**（跨机迁移必崩）。

- H4 每日自动化（08:00 跑 `h4_tracker.py`，计算/预警/H0 判定全自动，唯一人工=填表）。

## 3. 分阶段升级路线图

> 标记：【人】= 必须人工　【自】= 可自动化　【合规】= 受 D-08/09/11/15 约束

### 阶段 0 — 聚焦决策与止血（本周 / Week 0）
- **决策 P1**：明确"做宽(portfolio) vs 做深(PurePen)"。建议：**PurePen AI 为唯一营收引擎**，22 产品降为"展示型资产/交叉引流入口"，**停止新增产品**。
- **补备份 P5**：建立"本地 → GitHub(已是真相源) → 网盘/IMA"三级备份；**严禁备份 `.env*` / 含密钥文件**（`.gitignore` 已排）。
- **解锁死锁 P2**：今日起每日人工填 `h4-daily.csv`（唯一人工步骤）。

### 阶段 1 — 流量闭环跑通（Week 1–2）【H4】
- 【人】按 `h4-publish-calendar.md` 发小红书 11 篇（B 渠道，D-09 不挂链）。
- 【人】注册订阅号「论文降AI率」+ 挂 GH Pages 永久 hook URL（A 渠道，已部署 HTTPS）。
- 【自】H4 自动化每日 08:00 计算/预警/H0 判定。
- 里程碑：连续 3 天全达标 → 触发 🚦 H0 建议。

### 阶段 2 — 变现接通（Week 2–3）【H0】
- 【人+合规】按 `PRELAUNCH-CHECKLIST.md` 填 Waffo/Stripe 凭证 + Vercel 部署（**不自动烧凭证**）。
- PurePen AI 定价页 + 订阅流上线。
- 接真实 AI 率检测（朱雀/度小满）替换本地启发式（当前为启发式估算）。

### 阶段 3 — 技术债清理（Week 3–4）
- 修复/下线 3 个"构建中"产品；从 GitHub 110 仓库找回 ai-changelog-generator 对应源码或重建。
- 部署脚本**去硬编码**：路径改用环境变量/参数（`BASE_DIR`/`products_dir`/`vercel` 路径），消除跨机迁移风险。
- 统一"GitHub 真相源 ↔ 本地副本 ↔ Vercel" 流水线（复用 `auto_deploy_all.py`，先去硬编码）。

### 阶段 4 — 知识库驱动系统化（持续）【补 IMA】
- IMA 连通后，作为**统一知识中枢**：汇入 `项目索引.md` / `流水线手册.md` / `产品代码索引.md` / 各阶段结论 / 资产登记表。
- 固化"飞轮回填"：每次会话自动读 KB 而非重读索引；新交付物即时登记（资产表 v3）。
- `agent-orchestration-pipeline` 用于"内容批量生成 + 合规预检 + 发布清单"半自动闭环（已 TDD 收口）。

### 阶段 5 — 规模与增长（Month 2+）
- PurePen 验证 PMF 后，选 1–2 个高相关产品（writing/citation 类：`citeformatter` / `briefkit`）做交叉引流。
- GEO/SEO 内容持续（已有 30 天日历）；数据驱动校准 H4 阈值（7 日均值替换占位值）。
