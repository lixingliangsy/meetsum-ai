## 1. 转化闭环是什么（教学要点 + 三层模型）

转化闭环 = 从"被用户看见"到"最终成交"的完整路径。关键不只是**流量**，而是中间有没有**承接动作**（把流量沉淀到独立站并钩住）和**成交动作**（把沉淀变成订阅收入）。本案以「三层模型」组织：

- **触达层**：用户在哪被你触达（站外落地页 v3 / affiliate 站外承载 / 小红书种草标识 AI 不挂链 / 目录站与工具导航 Product Hunt·HN·Indie Hackers·SaaSHub）。四路口径均绕开 XHS 虚拟商品自营门槛，汇入同一承接—成交引擎。
- **承接层**：用户如何被接住（独立站免费注册 → 上传文档 → 多检测器 AI 率报告钩子[诚实标注误判风险] → 案例库对照展示语义保真改写 SPS>0.7 + 术语保护零丢失的可感知价值）。
- **成交层**：第一次付费如何发生（钩子检测报告免费/极低价 → 语义保真改写订阅[保真溢价主力，Waffo 收款] → 升单非母语/企业 → 复购）。

这一步直接决定你如何拿到**第一批用户与第一笔订阅收入**。对本案而言有一个特殊约束：当前闭环"工程就绪但未上线未起量"——Supabase 迁移 SQL 待执行、凭证未配（.env 仍含 REPLACE_ME）、Vercel 待部署、流量层未启动（H0 上线三缺口，P0 前置闸门）。所以**统一成交引擎（独立站订阅 + Waffo 收款）必须 H0 打通后才能跑真实流量与真实收费**，上线前只能跑"触达口径准备 + 承接引擎自测"。

因此本阶段给出**四路触达口径并行**结构（站外获客主轴，绕开 XHS 虚拟商品门槛），而非单线——四路都汇入同一个"承接—成交"引擎：

- **路径 A · 落地页 v3 触达口径**：自有站外落地页（含 ?tuid affiliate 承接位）直接引量到独立站，是主成交口径。
- **路径 B · affiliate 站外口径（D-09）**：小红书仅种草不挂链，转化放到允许外链的站外渠道承载 `?tuid` + 福利码，养号期即可并行赚佣金并反哺自有独立站。
- **路径 C · 小红书种草标识 AI 不挂链口径**：小红书仅种草（标识 AI / 降重前后对照），纯公域涨粉与信任背书，不挂链、不 AI 托管发布（D-08 半自动姿态）。
- **路径 D · 目录站与工具导航口径**：Product Hunt · HN · Indie Hackers · SaaSHub 等出海 SaaS 目录/社区首发，长尾 SEO + 开发者口碑获客，转化同样放站外独立站承载。

## 3. 路径 A · 落地页 v3 触达口径（主成交引擎）

### 适用条件
- 落地页 v3（含 ?tuid affiliate 承接位 + 自有独立站入口）已就绪，是承接 02「站外获客·部署衔接」的主口径。
- 适合：学术/专业写作者（主利基 02）直接被"降重降AI + 语义保真 + 合规去AI味"价值主张击中，经落地页直达独立站成交。

### 三段式路径
```
触达层：站外落地页 v3（自有公域承接位）+ 目录站收录 + 搜索矩阵
  ↓
承接层：独立站免费注册 → 上传文档 → 多检测器 AI 率报告（钩子，免费）→ 案例库对照展示保真价值
  ↓
成交层：语义保真改写订阅（免费检测→主力订阅）→ 升单非母语/企业 → Waffo 收款
```

### 漏斗节点与指标（对齐 06 漏斗 + H0/H1/H4/H5）
| 节点 | 指标 | 对应假设 | 可验时点 |
|---|---|---|---|
| 落地页曝光 → 访问（独立站） | 点击率、访问量 | **H4 站外冷启动** | 待 H0（流量层启动后） |
| 访问 → 免费注册/上传 | 注册转化率 | H4 获客漏斗 | 待 H0 |
| 上传 → 检测报告（钩子领取） | 钩子领取率、报告生成成功率 | H1 价值前哨 | 待 H0（含诚实误判标注自测现在可做） |
| 检测报告 → 订阅（保真 vs 纯降率选择率） | 保真订阅选择率 ≥50%（H1 成立线） | **H1 保真溢价** | 待 H0 后小样本 A/B |
| 订阅 → ARPU/LLM 成本 | LLM 成本/ARPU <30%（H5 成立线） | **H5 订阅单位经济** | 待 H0 首单回测 |
| 订阅 → 升单（非母语/企业） | 升单率、客单价 | H2（留待主利基跑通） | 主利基跑通后 |

### 流失监控点
- 曝光→访问弱：落地页价值主张未命中 / 搜索词未覆盖 → pivot（重设渠道组合 06 §4.2 B1，引入 D 路目录站/社区）。
- 访问→注册弱：承接摩擦（注册 UX / 报告钩子吸引力）→ 优化独立站承接。
- 报告→订阅弱（保真选择率<35%）：H1 不成立 → pivot A1 退守低价 / A2 强化保真感知（案例库对照演示 SPS 度量 / 术语零丢失，06 §4.1）。
- 订阅→单位经济弱（LLM 占比≥45%）：H5 不成立 → pivot C1 调价/改按量 / C2 优化 LLM 成本（06 §4.3）。
- 全链路起量失败（CAC>ARPU）：H4 不成立 → pivot B1/B2（借 01 人脉/内容杠杆降 CAC）。
