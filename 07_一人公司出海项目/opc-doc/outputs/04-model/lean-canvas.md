## 1. Problem（问题）

### Top 3 客户痛点（源自 03 价值主张 Pains）
1. **P1 · AI 率红线刚性且大**：本科 ≤20%、硕士 ≤15%、核心刊 ≤10%；过不了不能投、不能交、不能盲审/答辩。红线是硬约束，不是"想不想降"而是"必须降"。
2. **P2 · 通用改写/翻译工具破坏专业表达**：QuillBot（~$8.33/月）、Wordtune（~$13.99/月）等面向英文通用改写，中文学术语境水土不服、贵、与中国检测生态（知网/维普/朱雀）脱节。
3. **P3 · 纯降率黑产暴力洗稿**：为压 AI 率把语义改得面目全非、术语丢失，甚至承诺"绕过检测/不标注过审"，踩学术诚信与合规双红线。

### 伴生痛点（影响转化但非第一梯队）
- **P4 · 检测器口径不一**：6 个主流检测器一致性仅约 63%，改完此处过、彼处不过，焦虑且反复返工（改写后检测准确率亦骤降）。
- **P5 · 术语/专有名词被改坏**：核心名词被替换或丢失，专业度崩塌，导师/审稿人一眼识破。
- **P6 · 合规收紧后去AI味两难**：非母语写作者误判率高达 61.3%；2025-09-01《人工智能生成合成内容标识办法》施行后，既要降 AI 率又要主动合规标注，黑产方案直接违规。

### 现有替代方案及其缺口
| 现有替代 | 用户怎么解决 | 缺口（我们的切口） |
|---|---|---|
| 纯降率黑产 | 低价买"洗稿" | 破坏语义、丢术语、不合规（P3/P5/P6） |
| 国际改写工具（QuillBot/Wordtune） | 买英文通用改写订阅 | 中文脱节、贵、不针对中文检测生态（P2） |
| 检测方（朱雀/知网/度小满） | 只检不改 | 只给分数不给改写，价值网络分离 |
| 手动改 / Office 自助 | 自己改、耗时间 | 破坏专业度、慢、无保真度量 |

**核心缺口**：市场缺「保真 + 术语保护 + 合规」的可量化分层——把"降AI率"从黑产叙事拉回合规工程化。这正是 PurePen 的切口（语义保真 SPS>0.7 + 术语必含 + 合规去AI味）。

## 2. Solution（解决方案）

> 对应 03 价值主张 Value Map，以「检测报告建信任 → 语义保真改写 → 术语保护 → 合规去AI味」四段承接。P2 语义保真/术语保护/软降级**已实跑**。

| 产品/服务 | 角色 | 差异化封装 |
|---|---|---|
| 多检测器 AI 率报告（上传文档 → 报告） | 钩子 / 信任锚 | 首屏给"多检测器一致性 + 误判风险提示"，建信任、展示诚实（不承诺 100% 过检） |
| 语义保真改写（SPS>0.7） | 主差异化 · 核心 | 量化保真（全文 embedding 余弦 >0.7 高保真），区别于暴力洗稿；P2 已实跑 |
| 术语保护模块 | 护城河支点 | 专有名词必含校验，机构/理论/方法名零丢失；P2 已实跑 |
| 合规去AI味模块 | 合规护城河 | 依 2025-09 标识办法主动合规，不绕过检测、不承诺 100% 过检（软降级）；P2 已实跑 |
| 多检测器口径适配 | 降返工 | 针对知网/维普/朱雀主流口径统一优化，缓解 63% 一致性焦虑 |
| 非母语降误判场景包（备选②） | 向上细分升单 | 专门降误判策略，缓解 61.3% 高误判 |
| 企业泛化去AI味（备选③） | 泛化扩展升单 | 行业报告 / 智库稿件的合规去AI味泛化 |

**Solution 的护城河逻辑**：保真能力由代码量化（SPS）而非嘴说；软降级体现"可度量、可验证、不造假"的诚信差异化；合规去AI味从成本转护城河（2025-09 标识办法）。三者叠加 = 红海里唯一且可量化的分层。

## 4. Unique Value Proposition（独特卖点 / UVP）

**一句话 UVP**（继承 03 主推版）：
> 为需过知网/维普/朱雀 AI 率红线的学术与专业写作者，提供「语义保真（SPS>0.7）+ 术语保护 + 合规去AI味」的中文降重降AI SaaS——不只把 AI 率压下去，更把你的专业表达、专有名词和学术诚信原样保住。

**为什么不一样**：
- vs **纯降率黑产**：不破坏语义、不丢术语、不碰绕过检测黑产，在合规与学术诚信护栏内降率。
- vs **国际改写工具**（QuillBot/Wordtune）：专为中文学术语境训练、针对知网/维普/朱雀主流口径优化、内置术语保护，不与其英文赛道价格战。
- 一句话较长，钩子压缩与文案属 07 转化闭环，本阶段只锁定主张本身。

## 5. Unfair Advantage（不公平优势）

> 不可被竞争对手简单复制、且非"更努力就能追上"的优势。

1. **已实跑的代码资产（核心增量）**：PurePen SaaS（Next.js14 + Supabase + 部署链路）+ P2 语义保真/术语保护/软降级 + Supabase schema，可运行、可订阅——边际成本趋零（一次构建、无限服务）。
2. **创始人即典型用户**：学术写作（采矿沉陷 + LLM 综述投稿刚需）+ Hermes Agent 开发 + LLM/Embedding 集成，三者兼备且稀缺；需求同源（不是猜用户，是做给自己用）。
3. **科学计量学"可度量、可验证"思维已产品化**：SPS 量化保真、AI 检测未配置即"未度量"绝不编造——红海里清晰的质量/诚信差异化支点。
4. **合规去AI味护城河**：2025-09 标识办法后主动合规从成本转护城河；黑产（合规脆弱）与大厂（不敢碰/检测方只检不改）均难复制。
5. **平台链全通**：Supabase / Waffo / OpenAI 兼容 / Vercel / GitHub 一人公司友好，收款（Waffo Webhook）已通。
6. **【待核实 · 暂不作已确认护城河】人脉与内容影响力**：学术圈/开发者圈/AI 工具社群规模、学术成果/技术博客/案例库存量（01 资源 4 项待核实）——可作冷启动与扩展杠杆，但须创始人补齐信息方可计入。

## 7. Customer Segments（客户细分）

主利基人群：中文语境下的学术 / 专业写作者——硕博研究生、高校教师、科研人员、专业撰稿人（毕业论文、核心刊投稿、基金本子），需过知网/维普/朱雀 AI 率红线、但要求专业表达不被破坏。

| 子人群 | 位置（成稿→过检→提交链路） | 最痛场景 | 入口产品 |
|---|---|---|---|
| **A · 硕博 / 高校师生（最高频主入口）** | 成稿端 | 自己用 AI 辅助写，AI 率过高过不了，手动改又破坏专业度，盲审/答辩 deadline 紧 | 检测报告 + 语义保真改写 |
| **B · 科研人员 / 专业撰稿人** | 成稿端 | 术语密集、专有名词多，通用改写把核心概念改丢，需经得起同行审阅 | 术语保护 + 改写 |
| **C · 非母语学术写作者（备选②向上细分）** | 成稿端 | 原创被高误判（61.3%），申诉无据、最不公平 | 非母语降误判场景包 |

> 三类子人群痛点性质一致（要降 AI 率、又不能破坏专业度、还要合规），可用同一套"语义保真 + 术语保护 + 合规去AI味"组合承接，差异仅在场景包垂直颗粒度。
> 备选③ 企业/职场/自媒体（泛化扩展，六维 22，最红海）不作引流主轴，主利基跑通后承接。

## 8. Cost Structure（成本结构）

> SaaS 零物流、零库存，成本结构极轻，核心是"按量 API 成本 + 创始人双线精力 + 工程前置一次性投入"。

| 成本项 | 性质 | 说明 |
|---|---|---|
| LLM / Embedding 调用费 | 主要持续成本（按量） | PRD KPI：单用户 LLM 处理成本 < 30% ARPU；无物流/库存 |
| 云资源（Vercel / Supabase / Railway） | 持续按需付费 | 量级可控，SaaS 边际成本趋零 |
| 创始人时间（双线：学术 + 工程） | 主要持续成本 | 迭代 + GTM 内容产出 |
| Waffo 订阅（$9.9/月、$99/年） | 收款成本（已通） | 面向中国开发者收全球款 |
| 工程前置（一次性） | 凭证配置 + Supabase 迁移 + Vercel 部署 | 闭环从"工程就绪"到"可服务"的硬门槛 |
| **【依赖创始人补齐 · 待核实】** | 资金 / 精力 / 人脉 / 内容 | 初始资金规模→冷启动补贴与获客投入能力；精力区间→迭代与 GTM 节奏；人脉→冷启动零成本获客可行性；内容→信任背书与获客成本（详见 01 资源 4 项待核实） |

**成本特征**：无物流、无库存；边际成本趋零（一次构建、无限服务）；瓶颈在"凭证/部署/流量三缺口"与"创始人双线精力"（D-08 半自动姿态下人工目检保留）。
