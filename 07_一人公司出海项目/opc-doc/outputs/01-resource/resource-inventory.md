## 二、逐类明细

### 1. 技能与知识 ｜ 状态：可用 ｜ 杠杆度：高
- **学术写作领域知识**：西安科技大学采矿工程 / 开采沉陷 / 科学计量学方向背景，具备学术写作与论文发表经验。这构成 PurePen「学术风格降重 / 术语保护」功能的需求理解与质量基准——懂论文作者的痛点（术语不能乱改、引用须保留、语义须保真）。
- **AI Agent 开发技能**：主导 Hermes Agent 等 Agent 框架开发，具备从零搭建 Agent / 工作流系统的工程能力。这是 SaaS 后端 AI 推理编排、批量改写流水线的底层能力来源。
- **LLM / Embedding 集成经验**：已落地语义保真度量（全文 embedding 余弦相似度 >0.7 判高保真）、降重/降AI 改写 prompt 工程、AI 检测软降级逻辑。直接对应 PurePen 的核心算法资产（见第 6 类）。
- （pivot 后变化）原「小红书 2026 运营知识」「Python 自动化（Selenium/Playwright 上架）」「模板设计」三项技能，在 XHS 虚拟商品视角下为核心；pivot 后：① 小红书运营知识转为**站外 GTM 获客手段**（内容种草引流至站外 SaaS，非商品运营）；② Python 能力保留并升级为 **AI 推理微服务 / 文档解析（markitdown + python-docx + pdfplumber）**；③ 模板设计能力弱化，转为报告样式 / 案例库呈现。

### 2. 时间与精力 ｜ 状态：待核实 ｜ 杠杆度：中
- 已知事实：**一人公司运营**，且并行「学术线 + 工程线」双轨道（学术背景仍在，同时主导 PurePen 工程）。
- 创始人未披露每日/每周可投入 PurePen 的工时区间、双线精力分配比例、是碎片化还是整块时间。**记为待核实**。
- 作为关键约束变量记录：它直接决定产品迭代节奏、GTM 内容产出频率，以及是否需要引入外包/协作（见第 4、8 类）。

### 3. 资金与现金流 ｜ 状态：半可用（部分待核实） ｜ 杠杆度：中
- **SaaS 边际成本低（已确认事实）**：单位经济上 LLM 处理成本可控，PRD KPI 设定「单用户 LLM 处理成本 < 30% ARPU」；无物流/库存成本，无实物交付。
- **Waffo 收款链路已通（已确认）**：面向中国开发者收全球款，支持订阅+一次性，Webhook 自动通知，可提现至大陆银行卡/支付宝。收款侧不是瓶颈。
- 初始投入规模（启动资金、现金流压力、可用于外包/算力的预算）**未披露 → 待核实**。已知无重资产投入，主要成本为 LLM 调用费 + Vercel/Supabase/Railway 云资源费（按需付费，量级待核实）。

### 4. 人脉与关系 ｜ 状态：待核实 ｜ 杠杆度：中

### 5. 内容与影响力 ｜ 状态：待核实 ｜ 杠杆度：中

- **案例库内容资产（案例库 harvest）**：PurePen 已具备「案例库 harvest」能力，可沉淀真实降重/降AI 案例作为内容素材与模型样本。当前 harvest 规模、质量、是否公开**待核实**。

### 6. 产品与代码资产（核心增量） ｜ 状态：可用 ｜ 杠杆度：高
- **PurePen 代码库（已落地）**：Next.js 14（App Router）+ TypeScript + Tailwind + shadcn/ui 前端；Next.js Route Handlers 主后端 + Python FastAPI 微服务（AI 推理隔离）；Supabase（Postgres + Auth + Storage）数据层；Waffo 支付；OpenAI 兼容 LLM / Embedding 接入。整套 SaaS 可运行、可订阅。
- **P2 语义保真度量工程（已具备）**：全文 embedding 余弦相似度计算，阈值 >0.7 判「高保真」，量化改写后的语义保留程度——这是区别于普通洗稿工具的核心质量资产。
- **术语保护工程（已具备）**：用户指定术语（如专业名词、人名、公式符号）强制保留，不被同义替换破坏——直接服务学术/专业场景。
- **AI 检测软降级（已具备）**：朱雀 / 度小满等第三方 AIGC 检测 API 未配置时，系统返回「未度量」而非编造数字——体现「可度量、可验证、不造假」的产品伦理（见第 8 类）。
- **批量改写 / 案例库 harvest（已具备）**：支持批量文档处理与案例沉淀，为规模化与内容资产积累提供底座。
- **Supabase schema（D-17 接通）**：profiles / documents / rewrite_results / subscriptions 四表 + RLS 行级权限 + Auth + 2026 密钥兼容。数据模型已成型。
- **部署链路（已通）**：Vercel（前端/Next.js）+ Supabase 云 + Railway/Fly.io（Python worker），环境变量 + CI/CD 范式复用出海项目。
- **硬性产品边界（已定）**：D-11 降重须保原创性、反对学术不端、**不代写、只做生产引擎**（用户自用）；这是产品合规红线，也是品牌信任锚点。
- （pivot 后变化）原「小红书自动化引擎（~20 .py，含 P0 缺陷）」「ArgusBot 无人值守框架」「GitHub 私有库 ima-knowledge-base」：① 自动化上架引擎不再是核心交付链（XHS 商品上架已非主路径），但其 **compliance_gate 思路（限频/内容审核/滥用拦截）已复用为 PurePen 风控模块**；② ArgusBot 无人值守能力可转作 CI/部署兜底；③ GitHub 私有库保留，现承载 PurePen 代码中枢。

### 7. 平台与工具 ｜ 状态：可用 ｜ 杠杆度：高
- **Supabase**：Auth + Postgres + Storage + RLS，一人公司友好的后端一体化平台；已接通（D-17）。
- **Waffo（Pancake）**：支付与全球收款，替换原 open-saas 内置 Stripe；client/checkout/webhook 三件套已落地，Webhook 验签以官方 canonicalRequest 顺序为准（上线前核对）。
- **OpenAI 兼容 LLM / Embedding**：模型可切换（DeepSeek / Claude / GPT-4o-mini 等），成本可控；Embedding 用于语义保真度量。
- **Next.js 工具链**：Tailwind + shadcn/ui + Vercel 部署，SSR/SEO 利于获客；复用 12_Micro_SaaS 脚手架。
- **GitHub 私有仓库**：代码中枢（原 ima-knowledge-base 跨机共用账号）；另已克隆 GitHub awesome 系列（06-awesome-solo-founder-oss 等）作为 SaaS 工作流研究资料（非照搬代码）。
- **GTM 获客工具（半可用）**：小红书种草矩阵（路径 B，仅种草不挂链，D-09）、Product Hunt / HN / Indie Hackers / SaaSHub 等首发与目录（待 GTM 阶段启用）。
- （pivot 后变化）原「小红书个人店 / 千帆后台 / cookies-config-data」账号资产，在 XHS 视角为交易核心；pivot 后**降级为 GTM 种草触点**，不再承载店铺交易。千帆/cookies 等虚拟商品运营资产大部分闲置。

### 8. 独特经历与故事 ｜ 状态：可用 ｜ 杠杆度：中
- **学术 + 工程双背景（稀缺组合）**：既在采矿工程 / 开采沉陷 / 科学计量学方向做过真学术（懂论文作者痛点），又是从零主导 AI Agent 框架（Hermes Agent）的开发者——这种「能写论文也能写代码」的双语能力，在 AI 写作工具赛道里稀缺，是产品可信度与内容叙事的底层素材。
- **D-15 pivot 决策故事（真实转折）**：原小红书虚拟商品方向因「准入门槛封死」走不通，果断 pivot 到不受限的 SaaS（PurePen）。这个故事本身是 founder 叙事资产——体现「看清约束、快速转向、用工程能力自建解法」的创业者画像，可用于 build-in-public / 品牌故事 / 信任建立。
