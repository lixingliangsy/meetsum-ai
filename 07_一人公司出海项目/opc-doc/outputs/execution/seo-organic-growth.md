## ① pSEO 方法 + Google 2024 惩罚红线 + 三层安全框架（来自《程序化SEO完全指南》）

- **定义**：模板 + 数据批量生成大量关键词页面。案例：Zapier 80 万集成页、Wise 货币兑换对页、TripAdvisor 3 亿页。
- **Google 2024.3「规模化内容滥用」惩罚红线**：
  1. 每页 ≥ 60% 不同内容；
  2. 数据 ≥ 3 个独立来源；
  3. 提供搜索结果页之外的**独特价值**。
- **三层安全框架**：
  - 第一层 **数据底座要厚**：基础数据层 + 补充数据层 + 用户数据层交叉。
  - 第二层 **模板「变量深度」**：多层变量槽，非只改一个词。
  - 第三层 **人工/AI 审核**：质量 / 去重 < 40% / 价值检查。
- **实操路径**：找搜索模式 → 建数据库 → 设计模板（含 H1/区别/场景/步骤/FAQ/推荐）→ 批量生成（推荐 SSG 核心页 + SSR 长尾混合）→ 提交 sitemap + 内链 + GSC 监控。
- **三大坑**：数据太薄全套话 / 无用户价值只是占位 / URL 结构混乱（推荐 `/convert/png-to-jpg` 含关键词可读）。
- **认知**：pSEO 是**放大器非替代品**；先做小做对再做多；Google 要「每页有独特价值」非「每页都不同」。

## ③ PurePen 可落地的 pSEO 搜索模式举例（仅思路，不写页面）

| 搜索模式 | 模板思路（变量槽） | 数据层来源 |
|---|---|---|
| **{学术写作场景} + AI率检测** | `how to check AI rate in {literature review/thesis/essay}` | 基础=各检测器公开指标；补充=社区实测；用户=PurePen 检测对照（#28 案例库） |
| **{论文部分} + 降AI率工具/方法** | `lower AI rate in {conclusion/methodology/abstract}` | 同上 + 改写前后 AI 率对照 |
| **{检测器A} vs {检测器B} 对比** | `{Turnitin} vs {GPTZero} vs {ZeroGPT} accuracy & price` | 基础=官方准确率/价格；补充=用户实测；用户=PurePen 多检测器对照 |
| **{学术写作痛点} + 指南** | `how to avoid AI detection in academic writing without losing meaning` | 呼应 C 渠道 Reddit 高赞帖（#37 §4） |

- **模板变量槽（确保每页 ≥60% 不同）**：场景 / 检测器 / 语言 / 学科 / 痛点 / 步骤 / FAQ → 多层变量组合。
- **URL 结构**：推荐含关键词可读，如 `/guide/lower-ai-rate-{section}`、`/compare/{detector-a}-vs-{detector-b}`。
- **合规口径（D-11）**：所有页仅方法/工具分享，标注「非代写代做」，不承诺清零。
