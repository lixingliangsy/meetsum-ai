## 2. 零凭证钩子原理（为什么不需要凭证）

钩子的底层是 **PurePen 已实跑的本地检测 heuristic**（镜像 `src/lib/detect.ts` 的 `localHeuristic`，Python 侧即 `agent-orchestration/case-lib/local_ai_rate.py`）：

- 输入：用户粘贴的一段文本
- 算法：句长突发性（burst）+ 用字多样性（unique char ratio）→ 软信号估算 AI 率
- 输出：整体 AI 率 % + 风险等级 + **逐句高危段标红**
- 凭证：纯前端/本地计算，**零 LLM / 零 Supabase / 零 Waffo 凭证**

> 诚实标注：本地 heuristic 准确性有限（短文本/非母语噪声大），正式上线须接**朱雀 / 度小满 / 多检测器**为准。钩子页底部固定声明「本地启发式估算，非代写代做」。

已落地为自包含静态 HTML：`opc-doc/outputs/promo/a-channel/ai-rate-checker.html`（纯前端、零依赖、可静态托管，公众号菜单直接挂链到该 URL）。

## 3. 菜单 / 体验动线（智能体最小体验流程）

```
用户搜一搜「论文降AI率」 → 命中公众号 → 关注
  ↓
菜单①「免费测AI率」→ 打开 ai-rate-checker.html（粘贴段落 → AI 率初诊 + 高危段标红）
  ↓
菜单②「语义保真改写」→ 引导动作（见 §4，不在此步实跑 LLM 改写，避免凭证依赖）
  ↓
菜单③「术语不丢的降重」→ 方法分享 / 案例库对照（复用 h4-drafts 案例思路）
  ↓
观察指标：菜单点击 / 对话发起 / 连续追问 / 关注 / 收藏分享（喂 H4 追踪表）
```

**对话层（智能体最小体验）**：用户粘贴段落后，除报告外，可引导「回复关键词 **测AI率** 获取更多检测维度」或「回复 **预约** 加入语义保真改写等待列表」——均**不接单代做**（D-11），只做工具/方法分享与等待列表登记。
