## 一、漏斗模型与阈值（来自②）

```
自然曝光 ──▶ 点击 ──▶ 咨询 ──▶ 留资（关注 / 留邮箱 / 加微信 = 注册意向）
```

| 指标 | 阈值（校准值） | 说明 |
|---|---|---|
| 自然曝光 | 滚动 7 日均 ≥ **30** | 「稳定」判定窗口 7 天；数值为占位校准值，第 1 周结束后按实际调整 |
| 咨询数 | 滚动 7 日合计 ≥ **1** | 任一咨询即达标（>0） |
| 留资率 | 留资 ÷ 曝光 ≥ **1%** | 关注 / 留邮箱 / 加微信 ÷ 自然曝光 |

**H0 闸门触发条件**：上述三项**连续 3 天达标** → 生成 H0 上线建议（需人工确认凭证 + 部署，不自动执行）。
**重估条件**：连续 2–3 周零信号（曝光≈0 或持续 0 咨询）→ 暂停并重估钩子 / 词。

## 四、自动执行机制（四层逻辑）

### L1 · 数据录入层（人工 / 半自动）
- **人工录入**：每日把平台后台数字填进 `h4-daily.csv`（6 列：日期 / 渠道 / 曝光 / 点击 / 咨询 / 留资）。
- **半自动「拉取」**：把公众号 / 小红书专业号后台**导出的 CSV / JSON** 拖入目录，脚本读取并合并（非爬虫、非破解 API，合规）。
- **诚实边界**：小红书 / 公众号无开放公开 API，命令行**无法**无人值守直连抓取；真实「自动拉取」= 读取你下载的导出文件，或未来接**认证服务号 API**（需凭证）。发布动作永不在本层（D-08）。

### L2 · 自动计算层（全自动 · `h4_tracker.py`）
- 读取 `h4-daily.csv` → 按日期聚合 → 算留资率 → 滚动 7 日均值 → 逐项比对阈值 → 输出 `达标 / 未达标` + 连续达标天数。
- 纯 Python 标准库，无外部依赖，managed / system runtime 均可直接跑。

### L3 · 预警推送层（全自动）
- 异常检测：曝光日环比骤降 >50% → 限流 / 账号预警；连续 N 天 0 咨询 → 钩子失效预警；留资率连续下降 → 内容预警。
- 推送：写入 `h4-alerts.md` + 标准输出；预留 `WEBHOOK` 环境变量可接真实推送（企业微信 / 飞书 / 邮件）。

### L4 · 自动触发层（条件触发 · 需人工确认）
- 连续 3 天全达标 → 自动生成「🚦 H0 闸门触发建议」写入 `h4-status.md`。
- **仅建议，不执行**：H0 上线（填凭证 + 部署）仍需你 / 主理人确认（D-08 合规，不自动烧凭证）。

### 如何「无需人工干预」跑起来
1. 每日人工填 `h4-daily.csv`（**唯一人工步骤**）。
2. 用自动化每天定时跑 `h4_tracker.py`，计算 / 预警 / 触发判定全自动。
3. 你只需**看 `h4-status.md` 的状态标记**与预警，达标时确认上线。

```bash
# 定时运行（示例：每天早上 08:00，任意 Python 3.8+）
python "e:/AgentCPM/07_一人公司出海项目/opc-doc/outputs/execution/h4_tracker.py"
```
