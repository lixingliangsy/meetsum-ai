## 2. 瓶颈诊断（3 个假设 + 4. 我有自己的方案）

### 瓶颈假设①（最真实）：H4 零凭证验证方案未真正落地执行、未读到漏斗数据
- **适用情况**：② 已给方案 A/B/C + 漏斗阈值，但执行层（公众号注册 / 小红书发笔记 / 落地页）还没动，7 月毕业季窗口在流失。
- **依据**：2026-07-19 复盘已定 H4 为唯一可控瓶颈；② 进一步把 H4 从想法变成可执行的零凭证方案 + 决策门槛。需求已验证，卡点只剩「动手跑」。
- **代价**：若继续只规划不执行，7 月窗口不可逆流失，验证节奏后移，H0 闸门也因缺信号而迟迟不触发。

### 瓶颈假设②：H0 上线闸门前置错配 —— 凭证投入时机未对齐 ② 的决策门槛
- **适用情况**：当前 H0 被当作「等凭证就上」，但 ② 明确「核心词稳定自然曝光 + 咨询率>0 + 留资率>1% 才触发 H0」。若现在就投凭证上线，等于在「需求已验证但未验证我们能获客」前烧钱。
- **依据**：② 步骤 6 门槛。需求强 ≠ 我们能低成本获客；H0 凭证是一次性投入，应在 H4 信号通过后触发。
- **代价**：过早投入凭证 = 可能的沉没成本（若 H4 起量失败，已配 Supabase/LLM/Waffo 短期无回报）；但 H0 完全不动也错（它本就是 H4 通过后的承接）。

### 瓶颈假设③：资产复用链路未接上 ② 的搜索词/钩子（内容生产对不齐新关键词）
- **适用情况**：现有 E2 草稿生成器 / h4-drafts 用「降重前后 AI 率对比」母题，但 ② 推荐承接词是「论文降AI率 / 论文降重降AI / 去AI痕迹」，命名与钩子词需对齐，否则内容曝光命不中搜索词。
- **依据**：② 步骤 4 命名承接（用用户真实搜索词命名）。
- **代价**：内容若不带对搜索词，自然曝光门槛（② 继续开发门槛第一项）达不到，H4 信号失真，误判需求。
