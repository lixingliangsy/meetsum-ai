# M0.2 执行记录 · PurePen AI × Supabase

> 日期：2026-07-17｜主理人亲自实施并验收（不盲信 agent，沿用 P0 教训）

## 目标
接通 Supabase（Auth + Postgres + Storage + RLS），为 M1 降重/降AI 功能提供数据层与登录态。

## 完成清单（代码/配置层，全自动推进）

### 1. Schema（`supabase/schema.sql`）
- 4 张表：`profiles` / `documents` / `rewrite_results` / `subscriptions`
  - 订阅字段用 **`waffo_*`**（非 Stripe），webhook 写入
  - `rewrite_results` 含 `ai_score_before/after`、`similarity`、`style` —— 对应降重/降AI 核心指标
- `handle_new_user` 触发器：注册自动建 profile
- **RLS 修正**（按 2026 调研）：所有 policy 加 `to authenticated` + `auth.uid() IS NOT NULL`，防匿名绕过
- `is_paid_member(uid)` helper：付费门控（Pro 功能 RLS/逻辑用）
- `subscriptions` 仅开放客户端 `select own`——**写入只走 Waffo webhook 的 service-role key**，构成付费墙 defense-in-depth
- `docs` storage 桶 + 3 条 storage RLS（上传/查看/删除 own）
- `increment_document_usage()` 触发器：M0.4 计量钩子预埋

### 2. Auth 链路（Next.js 14 App Router，@supabase/ssr）
| 文件 | 作用 |
|---|---|
| `src/lib/supabase/server.ts` | `createServerClient`（带 cookie）+ `createServiceClient`（service role，webhook 用） |
| `src/lib/supabase/middleware.ts` | `updateSession()` 每次请求刷新 token cookie |
| `src/middleware.ts` | matcher 挂载刷新（排除静态资源） |
| `src/app/auth/auth-provider.tsx` | 客户端 `AuthProvider` 上下文（暴露 session/user/plan） |
| `src/app/auth/callback/route.ts` | `exchangeCodeForSession` 回调 |
| `src/app/auth/signout/route.ts` | 退出登录 |
| `src/app/login/page.tsx` | 邮箱魔法链接登录页 |

### 3. 密钥兼容 2026 命名
Supabase 2026 起弃用 anon/service_role，改用 **publishable (`sb_publishable_`) / secret (`sb_secret_`)**。
`client/server/middleware/callback/signout` 全部用
`NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY ?? NEXT_PUBLIC_SUPABASE_ANON_KEY` 双命名兼容。

### 4. 一键脚本
- `scripts/apply-schema.mjs`：`SUPABASE_DB_URL` 直连执行 schema.sql
- `scripts/validate-schema-local.mjs`：本地 Postgres + auth/storage 桩验证
- `scripts/validate-schema-parse.mjs`：pgsql-parser 静态语法验证
- `package.json` 加 `db:apply` / `db:validate:*` scripts
