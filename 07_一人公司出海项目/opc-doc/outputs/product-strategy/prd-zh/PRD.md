## 0. 产品一句话定位

**净文 AI** = 上传文档 → AI 降重 + AI 生成率（AIGC）检测 + 逐句改写建议 → 导出「低重复、低 AI 味」成稿的 Web 订阅制工具。
目标用户：毕业生、学术写作者、自媒体、需过 AI 检测的企业文案。**自建 SaaS + 站外转化，绕开小红书虚拟商品 180 天准入门槛**（D-15），正踩 7 月毕业季降重需求高峰。

## 1. MVP 范围定义

### 1.1 核心功能模块

| 模块 | 功能 | MVP 是否包含 |
|---|---|---|
| M1 文档接入 | 上传 docx / pdf / txt；解析为正文+段落结构 | ✅ |
| M2 降重引擎 | LLM 同义替换 + 句式重构 + 语义保留（学术/通用风格可选） | ✅ |
| M3 AI 检测 | 处理前/后 AI 生成率（%） + 高风险段落标注 | ✅ |
| M4 改写建议 | 原文↔改写逐句 diff + 一键采纳/局部应用 | ✅ |
| M5 订阅支付 | 免费额度 + 会员订阅（Stripe / 微信·支付宝） | ✅ |
| M6 报告导出 | 导出 Word / PDF（含修改痕迹） | ⚠️ MVP 后置（P2） |
| M7 历史/版本 | 处理历史、版本对比 | ⚠️ MVP 后置 |
| 批量/API/团队协作/插件/多语言 | — | ❌ 不在 MVP |

### 1.2 用户角色

| 角色 | 权限 | 说明 |
|---|---|---|
| 免费用户 | 每日/每月有限额度（如 3 篇/月，单篇 ≤ 3000 字） | 试用转化来源 |
| 付费会员 | 高额度/无限篇、降AI 模式、优先队列、导出 | 收入来源（月费 ¥29.9 起） |
| 创始人（Admin） | 模型配置、风控、用量计量、套餐 | 一人公司即本人 |

### 1.3 关键用户流程

```
[注册/登录] → [上传文档] → [选模式:降重/降AI/双模]
   → [AI 处理(流式)] → [查看报告:AI率前后对比 + 段落风险]
   → [逐句采纳改写建议] → [导出/复制成稿]
   → (额度不足) → [订阅套餐] → [支付] → [权益生效]
```

### 1.4 MVP 边界（IN / OUT）

- **IN**：单文档处理、降重+降AI、报告、基础订阅、Web 端、中文。
- **OUT（后续迭代）**：批量处理、开放 API、团队协作、多语言、移动端 App、浏览器插件、对接知网等查重库（需授权）、数字人/视频生成。

## 2. 技术栈选型

> 复用「AI 编程出海项目」工作流体系（见 §5），优先沿用 Next.js + Supabase + Stripe + Vercel 范式。

| 层 | 选型 | 理由 / 复用 |
|---|---|---|
| 前端 | **Next.js 14（App Router）+ TypeScript + Tailwind + shadcn/ui** | 复用 12_Micro_SaaS 脚手架；SSR/SEO 利于获客 |
| 后端 | **Next.js Route Handlers（主）+ Python FastAPI 微服务（AI 推理）** | 全栈减运维；重 AI 任务隔离到 Python worker |
| 数据库 | **PostgreSQL via Supabase**（Auth + Storage + RLS + 向量可选） | 一人公司友好，含 Auth/存储/行级权限 |
| AI·降重/改写 | **LLM API（DeepSeek / Claude / GPT-4o-mini）**，prompt 工程 + 流式 | 成本可控；DeepSeek 用于选题/改写，Claude 保质量 |
| AI·检测率 | **AIGC 检测 API（朱雀/度小满等）+ 自建启发式（困惑度/突兀度）兜底** | MVP 先用第三方 API + 自建基线，避免单点依赖 |
| 文档解析 | **markitdown + python-docx + pdfplumber**（复用 markitdown MCP） | 用户已有 markitdown 能力，零新增 |
| 导出 | **python-docx（Word）+ reportlab（PDF）** | 复用学术内容引擎经验 |
| 支付 | **Stripe（出海）+ 微信支付/支付宝（国内）** | 复用 12_Micro_SaaS 支付模块 |
| 部署 | **Vercel（前端/Next.js）+ Supabase 云 + Railway/Fly.io（Python worker）** | 复用出海部署范式；环境变量 + CI/CD |
| 风控 | **复用 compliance_gate 思路**：限频、内容审核、滥用拦截 | 见 §5 |

## 3. 验证指标（KPI）与 MVP 成功标准

### 3.1 核心 KPI

| 指标 | 定义 | MVP 目标 |
|---|---|---|
| 降重率 | (处理前相似度 − 处理后相似度) / 处理前 | 相似度降至 < 15%（对接查重/自建 shingles） |
| AI 检测通过率 | 处理后 AI 率 < 阈值（如 20%）的文档占比 | ≥ 70% |
| 7 日留存 | 注册后 7 日回访比例 | > 30% |
| 30 日留存 | 注册后 30 日回访比例 | > 15% |
| 免费→付费转化 | 付费用户 / 注册用户 | > 5% |
| 单位经济 | 单用户 LLM 处理成本 vs 月费 ARPU | 成本 < 30% ARPU |
| 处理量 | 日活跃处理文档数 | > 50（上线 30 天后） |

### 3.2 MVP 上线后成功标准（退出验证）

- **上线 30 天内**达到下列任一项即视为 MVP 验证通过：
  - 付费用户 ≥ 100 人；**或**
  - MRR（月经常性收入）≥ ¥3,000；**或**
  - 自然流量（小红书种草引流）日均注册 ≥ 20 且 7 日留存 > 30%。
- 未达标则进入迭代（优化降重质量 / 调整定价 / 强化获客）。

## 5. 参考文章方法论复用（toutiao 7662039148701696514）

参考文章《小红书电商2026最新红利：0粉开店+AI批量产内容，有人3月卖80万》的核心打法，映射为本 SaaS 的**获客工作流**（SaaS 自身在小红书只做种草、成交走站外，符合 D-08/D-09 渠道拆分）：

| 文章方法 | 本 SaaS 复用方式 |
|---|---|
| **数据化选品**：蝉妈妈按「增速>50% 且 笔记数<5000」找蓝海 | 用小红书搜索下拉词 + 评论高频问题定**内容选题**与**功能优先级**（如"降AI率""论文降重"为高需词） |
| **AI 内容流水线**：DeepSeek 选题→爆款模板框架→人工填真实体验（AI七人工三）→Canva AI 配图→即梦 AI 视频→剪映 | 复用为 SaaS 的小红书种草生产流；**必须保留真实人类参与**（去 AI 味 + 满足 04-27 治理主张，避免被判 AI 托管封禁） |
| **发布节奏**：日更 5–8 条（3 图文 + 2 视频），晚 7–9 点，前 30 分钟评论互动 | SaaS 官方号养号节奏；前两周数据一般属正常，第三周易爆 |
| **矩阵放大**：单号跑通复制 3–4 矩阵号，AI 重新生成避免重复 | 多账号种草矩阵（不同细分：毕业生/自媒体/职场），内容 AI 重写去重 |
| **变现三路径**：站内佣金 / 私域钩子（"评论扣1领攻略"）/ 商单（≥1000 粉蒲公英） | SaaS 采用：站内仅种草（不挂链）→ 私域/落地页承载注册 → 站外 SaaS 成交 |
| **A/B 测试**：同产品 3 版主图文案跑 3 天留最优 | 种草笔记封面/文案 A/B，提升点击→注册转化 |
| ⚠️ **数字人无人直播**（文章称增量 5 万/月） | **不采用**：04-27 治理主张将"AI 托管运营、无真实人类参与"列为违规运营（封禁级），与 D-08 半自动冲突。获客必须真人参与 + AI 辅助 |

## 6. 合规与风险（贯穿 MVP）

1. **平台合规（小红书获客端）**：种草笔记须标"AI 生成"（04-27）；不挂外链/二维码（D-09）；保持真实人类参与（D-08）。
2. **产品合规（SaaS 本身）**：降重须保原创性，反对学术不端（D-11）；不代写、只做生产引擎（用户自用）。
3. **风控**：限频防滥用、内容审核防侵权（复用 compliance_gate 模式）。
4. **数据合规**：用户文档不留存超期、可选自动删除；明示隐私政策。
