## M1 — 降重/降AI 核心功能（产品核心价值）
- `src/lib/detect.ts` + `detect-types.ts`：检测引擎抽象，**朱雀**（第三方 API `api.zhuqueai.com/v1/detection`，Bearer key，URL 可配）/ **度小满**（商务合作 API，填 URL+Key 启用）/ **本地启发式**回退；设计=失败软降级（未配置返回 score=null，不误导）。
  - 调研结论：朱雀有可调 API；**度小满 AI 文本检测为 B2B 商务合作（需签保密协议），无公开自助 Key**，故仅留适配器形状。
- `src/lib/llm.ts` 增强：`rewriteText` 风格化（学术/通用/精简）改写 + `estimateAiScore` 模型自估（无检测引擎时的软信号）。
- `src/lib/similarity.ts`：基于归一化编辑距离的改写前后相似度。
- `src/app/api/rewrite/route.ts`：鉴权 → 配额 → 改写前 AI 率 → LLM 改写 → 改写后 AI 率（真实引擎优先，否则模型自估）→ 落库 → 返回前后 AI 率/相似度/引擎；`maxDuration=60`。
- `src/app/api/detect/route.ts`：独立检测端点（不落库、不计配额）。
- `src/app/dashboard/page.tsx`：核心 UI（原文/结果双栏、风格选择、AI 率前后对比条、相似度、用量徽标、复制、退出）。
- `src/app/pricing/page.tsx` + `src/components/checkout-button.tsx`：会员方案页 + Waffo 收银台拉起（落地页原本链接 /dashboard 与 /pricing，现已补齐）。

## 仍缺（待用户补充，不影响构建）
1. **Supabase URL / publishable / secret** —— `.env.local` 为占位符；远端 DDL 需 `npm run db:apply`（填 SUPABASE_DB_URL）。
2. **LLM_API_KEY** —— 占位；无 Key 时改写接口会报 internal_error（运行时缺凭证）。
3. **Waffo 产品 ID** —— `src/lib/waffo.ts` 的 `waffoProductId` 仍为 `PROD_REPLACE_ME`，需在 Waffo 后台建产品后回填，否则 checkout 失败。
4. **朱雀/度小满检测 Key** —— 未配时 UI 显示"本地估算/未配置"，接引擎后自动升级为真实 AI 率。

## M1.1 流式 / 批量 / 报告导出（2026-07-17 续做，全自动化）

### M1.1a 流式改写（SSE）
- `src/lib/llm.ts`：新增 `streamRewrite(text, style, onToken?)` —— OpenAI 兼容 `stream:true`，逐片回调 `onToken`，最终返回完整文本。未配 `LLM_API_KEY` 抛错由路由转译 `internal_error`。改写系统提示词抽为 `rewriteSystemPrompt()` 单篇/流式共用。
- `src/app/api/rewrite/stream/route.ts`（新增）：SSE 事件流 `before → chunk* → after → done`，失败 `error`。鉴权 + 配额校验 + 改写前检测 + 流式改写 + 改写后检测 + 落库，全链路一致。
- **TS 修复**：新版 lib 的 `Uint8Array<ArrayBufferLike>` 不再直接兼容 `Response` body 类型 → `sse()` 改返回 `string`，`controller.enqueue` 处用 `toBytes()` 包成 `Uint8Array`。

### M1.1b 批量降重
- `src/app/api/rewrite/batch/route.ts`（新增）：接收 `items[]`，逐条 配额→改写前检测→改写→改写后检测→相似度→落库；单条超额标记 `quota_exceeded`、失败不影响其余；批量上限 20 篇。

### M1.1c+d Dashboard 重构
- `src/app/dashboard/page.tsx`（重写）：
  - 单篇模式：改用 SSE 流式渲染（fetch + `response.body.getReader()` 解析 SSE 块，逐字显示）。
  - 批量模式：文本用 `====` 分隔多篇，结果以卡片表展示（前/后 AI 率、相似度、复制）。
  - 报告导出：单篇 + 批量均支持 Markdown 报告（前后 AI 率/相似度/引擎/原文/改写后），`Blob` 下载 `.md`，纯客户端。
  - 保留 402 跳转定价页、退出、合规说明。

### Waffo 配置守卫

## 小红书起号方案（2026-07-17 联网调研）
- 落盘：`opc-doc/outputs/growth/xiaohongshu-launch-plan.md`
- 调研依据：2026 小红书起号方法论、AI 工具类账号打法、《人工智能生成合成内容标识办法》（2025-09-01 施行）、小红书 AI 治理主张（即 D-08 红线）。
- 核心要点：
  1. **定位**：净文 AI = 「帮你把文字改得更自然、更像真人写的」合规润色助手。**绝不**教人"绕过 AI 标识/不标注过审"（违反标识办法 + 平台红线 + 产品不代写/不绕检测原则）。
  2. **账号搭建**：昵称含关键词、统一视觉、简介一句话价值 + 引导官网。
  3. **内容支柱**：引流款 60% / 专业款 30% / 人设款 10%；覆盖降 AI 率技巧、朱雀/度小满科普、AI 标识合规教程。
  4. **搜索流量为主**（约 70%）：长尾词布局「降 AI 率 / 论文降重 / AI 标识 / 学术润色」。
  5. **首 15 篇选题日历**：对比型/数字型/痛点型封面，20:00–22:00 发布，前 15 篇高度垂直钉标签。
  6. **合规护栏**：AI 内容必勾「笔记含 AI 合成内容」；不冒充真人/批量伪造；私信留微信=引流=限流→用主页/置顶评论引导搜官网。
  7. **转化闭环**：小红书(价值钩子) → 官网免费版试用 → Pricing(Waffo Pro $9.9/$99) → 私域仅作会员服务不硬引流。
- 本方案属执行期运营动作（07 转化闭环在小红书渠道落地）；如需系统化触发「资产沉淀（沉墨）」固化选题库/封面模板/复盘 SOP。
