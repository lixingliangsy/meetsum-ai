## 1. 最该优先验证的假设：H0（P0 前置）+ H1 / H4 / H5（P1 主验证）（单一主验证目标）

### 1.1 为什么是这些（而非 H2/H3）

来自 `04-model/risky-assumptions.md` 与 `lean-canvas.md` 的假设队列，按"是否能在主利基上线后立即拿到真实市场信号"筛选：

| 假设 | 优先级 | 本阶段能否验证 | 处置 |
|---|---|---|---|
| **H0 上线前置三缺口** | P0 | ✅ 能（上线前必先打通，否则无用户无收入） | **前置闸门 · 最先** |
| **H1 保真溢价接受度** | P1 | ✅ 能（上线后小样本 A/B 即测） | **主验证 · 价值侧** |
| **H4 站外获客冷启动** | P1 | ✅ 能（上线后漏斗监测即测） | **主验证 · 获客侧** |
| **H5 订阅单位经济/价格锚** | P1 | ✅ 能（上线后首单回测即测） | **主验证 · 收入侧** |
| H2 细分扩展可迁移性 | P1 | ❌ 需主利基先跑通才有扩展基数 | 留待主利基跑通后 |
| H3 合规护城河 | P1 | ❌ 需留存/复购数据 | 留待主利基跑通后 |

判定逻辑：
- **H0 是上线闸门**：当前闭环"工程就绪但未上线未起量"——Supabase 迁移 SQL 待执行、凭证未配（.env 仍含 REPLACE_ME）、Vercel 待部署、流量层未启动。四者任一卡住，订阅收入无从谈起。故 H0 须最先打通，是后续一切验证的物理前提（与旧版 H0 资质的"时间闸门"等价，但本版为部署闸门）。
- **H1/H4/H5 是上线后即可并行拿信号的三项 P1**：一价值侧（溢价是否有人买单）、一获客侧（站外能否引量）、一收入侧（单位经济是否成立）。三者共同回答"产品上线后商业是否成立"，是 MVP 最小验证闭环的主干。
- **H2/H3 本质是主利基跑通后的扩展/留存假设**：无扩展基数与留存数据前强验会污染信号，留待主利基跑通后由 07 承接（见 §5）。
- **P2 能力已实跑收敛**（语义保真 SPS>0.7 / 术语保护 / 软降级），不占验证队列——MVP 不验证"能不能做出来"，只验证"做出来后市场买不买账、单位经济成不成立"。

### 1.2 单一主验证目标（把 H0 + H1/H4/H5 合成一句话）

> **"PurePen 能否先打通 H0 上线三缺口（部署+凭证+流量启动），让'上传文档→多检测器 AI 率报告 + 语义保真改写(SPS>0.7) + 术语保护'的已实跑能力面向真实用户可服务可收费，并在小样本下验证 H1 保真溢价有人买单、H4 站外能低成本引量、H5 订阅 ARPU 能覆盖 LLM 成本（<30%）？"**

三者耦合：H0 提供"可被验证的物理环境"，H1 验价值是否被认可，H4 验流量是否可获取，H5 验单位经济是否成立。任一不成立，SaaS 主路径需重估（见 §4）。

## 2. 最小验证实验设计（针对 H0 + H1 + H4 + H5）

### 2.1 核心 MVP 产品形态（已实跑，验证对象本身）

> 上传文档 → ① 多检测器 AI 率报告（诚实标注：6 检测器一致性仅约 63%、非母语误判率 61.3%、改写后检测准确率骤降，故报告须含"误判风险提示 + 软降级不造假"）→ ② 语义保真改写（SPS = embedding 余弦 > 0.7，2026 新指标，与 PurePen P2 阈值对齐）→ ③ 术语保护（专有名词必含校验，零丢失）。**以上为代码已实跑能力，本 MVP 直接复用，不重造。**

### 2.2 最小动作（按优先级排队）

1. **H0 前置闸门（最先）**：执行 Supabase 迁移 SQL → 配齐 LLM / Waffo / Embedding 凭证（.env 去 REPLACE_ME）→ Vercel 部署公网可访问 → 站外流量层启动（落地页/affiliate/内容）。验证"可服务可收费 + 首批曝光"。
2. **H1 保真溢价 A/B（上线后小样本）**：对首批访问用户给两组入口——"保真订阅（语义保真+术语保护+合规）" vs "纯降率低价"——测选择率与付费转化差（对标 QuillBot ~$8.33/月、Wordtune ~$13.99/月，中文适配弱，仅参照不照搬）。
3. **H4 站外冷启动漏斗（与 H1 并行）**：监测"站外曝光（落地页/affiliate/小红书种草不挂链）→ 独立站访问 → 注册/上传 → 检测报告 → 订阅"全链路转化与单获客成本（CAC）。
4. **H5 订阅单位经济（首单回测）**：用真实首单 ARPU 与 LLM 处理成本对账，测算 LLM 成本占 ARPU 比是否 < 30%（PRD KPI）。

> 注意：这是"验证"而非"经营"。最小动作 = 复用已实跑产品 + 打通部署/凭证（清单级，实现由工程线推进）+ 上线后跑小样本 A/B 与漏斗 + 首单单位经济测算。不写文案、不进实现细节。

### 2.3 所需素材（明确可用既有成品，零新造能力）

| 素材 | 来源（既有） | 在实验中的角色 |
|---|---|---|
| 语义保真改写 / 术语保护 / 软降级引擎 | 01 资源确认 `src/lib` 已实跑 | 核心 MVP 能力，直接复用 |
| 多检测器 AI 率报告模块 | 已实跑（含误判风险提示/软降级） | 钩子 + 诚实定位载体 |
| 案例库（保真改写前后对照） | 01 资源已生成 | 信任背书、保真感知价值演示 |
| Supabase 迁移 SQL / 部署配置 | 工程就绪待执行 | H0 部署闸门清单 |
| 站外落地页 v3 / affiliate 链路 | 02/03/07 衔接产物 | H4 冷启动获客载体 |
| 定价梯度框架（不写数字） | `04-model/pricing-notes.md` | H1 保真溢价 A/B 的梯度依据 |

### 2.4 验证周期建议（上线闸门 → 小样本 → 中期判定）

节奏受 01 资源 4 项待核实约束（时间/资金/人脉/内容），以下为理想节拍，实际以主理人补齐信息后校准：

| 阶段 | 周期 | 重心 | 最小节奏 |
|---|---|---|---|
| **闸门期（H0）** | 上线前最短路径 | 打通部署+凭证+流量启动 | 执行迁移 SQL → 配凭证 → 部署 → 启动流量层（依赖 01 时间与资金） |
| **探针期（H1/H4/H5）** | 上线后 ~4–6 周 | 小样本 A/B + 漏斗监测 + 首单单位经济 | 每日/周监测曝光→订阅→ARPU/LLM 成本，不追量 |
| **中期判定点** | 第 6 周 | 判定 H1/H4/H5 是否成立 | 三假设达阈值则进放量，不达则按 §4 pivot |

## 3. 验证形式与度量（指标 + 阈值 + 判定线）

### 3.1 验证形式

按假设分四类最小形式：

| 假设 | 最小验证形式 |
|---|---|
| **H0** | 部署可用性测试 + 凭证打通验证 + 流量层启动监测（首单/首批曝光） |
| **H1** | 定价梯度小样本 A/B（保真订阅 vs 纯降率低价），测选择率与转化差 |
| **H4** | 漏斗监测（站外曝光 → 独立站/订阅转化）与单获客成本（CAC） |
| **H5** | 单位经济测算（真实 ARPU vs LLM 处理成本，目标 < 30%） |

### 3.2 漏斗与指标体系

```
站外曝光(落地页/affiliate/小红书) → 独立站访问 → 注册/上传 → 检测报告 → 订阅转化
                                                          ↑H1 保真溢价(选保真订阅比例)
                              ↑H4 获客(CAC/转化率)        ↑H5 单位经济(ARPU vs LLM成本<30%)
```

| 维度 | 指标 | 健康阈值（成立线） | 预警线 | 不合格线 |
|---|---|---|---|---|
| **H0 上线** | 部署可用性（公网可访问+核心流程跑通） | ✅ 通过（含凭证全配齐） | 部分缺口 | 任一缺口未补 |
| | 站外流量启动 | 首批曝光 > 0 + 首单/首试用产生 | 曝光>0 但无转化 | 零曝光 |
| **H1 价值** | 保真订阅选择率（vs 纯降率低价） | ≥ 50%（保真组占多数） | 35–50% | < 35% |
| | 保真组付费转化差 | 保真组转化 ≥ 纯降率组（抗价差） | 低 10–20% | 低 > 30% |
| **H4 获客** | 落地页访问 → 订阅转化率 | ≥ 2%（小样本下限） | 1–2% | < 1% |
| | 单获客成本 CAC | CAC < 首年订阅 ARPU 回收期可接受（依赖 01 资金） | 接近 ARPU | CAC > ARPU |
| **H5 经济** | LLM 成本 / ARPU | < 30%（PRD KPI） | 30–45% | ≥ 45% |
| | 订阅毛利率（ARPU − 边际成本） | > 0 且正向 | 临界 | < 0 |

> 注：小样本期（~4–6 周）指标仅作方向性判定，最终数字由 07 转化数据回测定稿；本阶段不落定价数字。对标参照：QuillBot ~$8.33/月、Wordtune ~$13.99/月（中文适配弱，不照搬）。

### 3.3 假设成立 / 不成立判定线

- **H0 成立**：部署公网可用 + 凭证全配齐 + 站外流量启动 + 产生首批曝光/首单。
- **H0 不成立（阻塞）**：任一缺口未补 → 维持工程态无收入，须先补齐再验其余，不跳过。
- **H1 成立**：保真订阅选择率 ≥ 50% 且保真组付费转化不显著低于纯降率组（差 < 20%）。
- **H1 不成立（触发 pivot）**：保真组选择率 < 35% 或转化差 > 30% → 执行 §4.1。
- **H4 成立**：落地页→订阅转化率 ≥ 2% 且 CAC < ARPU 回收期可接受。
- **H4 不成立（触发 pivot）**：转化率 < 1% 或 CAC > ARPU → 执行 §4.2。
- **H5 成立**：LLM 成本 / ARPU < 30% 且毛利率正向。
- **H5 不成立（触发 pivot）**：LLM 占比 ≥ 45% 或毛利为负 → 执行 §4.3。
- **H0 + H1/H4/H5 全成立** → 进入放量 + 启动 H2/H3 深度验证（07）。
- **任一 P1 不成立** → 执行 §4 pivot，不阻塞 H0 但必须切换打法。

## 4. 若 H1 / H4 / H5 不成立：3 个备选 pivot 方向

> pivot 与主路径可并行小流量测试，但主路径连续不达标必须切换，不恋战。

### 4.1 H1（保真溢价）不成立的 pivot
- **Pivot A1 · 退守低价走量**：保真溢价不被买单时，先以接近纯降率的价格走量验证规模，再图升单。
- **Pivot A2 · 强化保真感知价值**：用案例库做"视频演示 SPS 度量 / 术语零丢失对照"提升保真可感知价值，重测溢价接受度（不改变能力，只改变沟通与证据呈现）。

### 4.2 H4（站外冷启动）不成立的 pivot
- **Pivot B1 · 重设渠道组合**：降低单一落地页依赖，引入小红书种草不挂链 + affiliate 双轨，或借 01 人脉圈（学术/开发者/AI 工具）做零成本种子扩散。
- **Pivot B2 · 引入内容/人脉杠杆**：用 01 内容影响力（学术成果/技术博客/案例库）做信任背书降获客成本；若人脉/内容待核实项偏弱，先补强再放量。

### 4.3 H5（单位经济）不成立的 pivot
- **Pivot C1 · 调价或改结构**：ARPU 不足覆盖 LLM 成本时，上调订阅锚或改"按千字/按篇一次性"付费结构（对应 `pricing-notes` 方案 B），降低 LLM 成本敞口。
- **Pivot C2 · 优化 LLM 成本**：切换更优模型/缓存/批量改写降边际成本，把 LLM 占比压回 < 30%。

## 5. 边界声明与留待后续

- **不涉及工程实现细节**：本 MVP 产品能力（保真改写/术语保护/软降级/多检测器报告）已实跑，仅列 H0「上线三缺口」部署/凭证清单作为验证前提；具体实现由工程线推进，本验证层不进代码细节。
- **不写获客文案/日程**：落地页正文、小红书种草文案、发布时间表属 **07 转化闭环**，本阶段仅定义"验什么/怎么验"与漏斗指标。
- **不落定价最终数字**：月度/年度价、按千字价、升单加价幅度由 06 小样本 + 07 数据回测定稿（见 `pricing-notes`）。
- **H2/H3 留待主利基跑通后**：H2 细分扩展（非母语/企业升单）、H3 合规护城河（留存/复购）需主利基有基数与留存数据后方可深验，不混入本窗口，避免信号污染。
- **合规红线（零容忍）**：依 2025-09-01《标识办法》，主动合规去AI味为护城河；不为降率踩合规线，检测报告诚实标注误判风险（6 检测器一致性 ~63%、非母语误判率 61.3%），软降级不造假。
- **01 资源 4 项待核实标注**：时间/资金影响 H0 上线速度与 H4 冷启动补贴能力；人脉/内容影响 H4 获客成本与 H5 早期收入——均影响验证节奏与爬坡速度，但不阻断 H0–H5 模型结构本身。

## 6. 人机分工（基于 D-15 半自动辅助姿态 + 合规红线）

| 角色 | 职责 |
|---|---|
| **人工（主理人）** | H0 工程前置决策（凭证/部署把关、上线拍板）；H1 保真溢价 A/B 梯度拍板；H4 渠道组合判断；合规把关（标识办法零容忍）；定价最终决策；pivot 决策；种子用户承接 |

| **边界** | AI 产出（尤其合规表述/定价沟通）须经人工目检后对外；上线与合规把关的人工决策不可省 |
